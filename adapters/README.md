# Lumi feed adapters — light up the overlay from any AI coding tool

The Lumi **overlay** and **VS Code panel** render lesson events from a shared feed at
`~/.lumi/feed.jsonl`. The feed is filled by `lumi feed --source <tool>`, which reads a tool's
output, detects new concepts, and appends lessons (generated by **that tool's own model** —
`--source codex` uses Codex, `--source gemini` uses Gemini, others default to Claude, all with an
offline mock fallback so it never hard-fails).

Each tool has a different "after the AI responds" hook. The adapters below wire each one to
`lumi feed`. They mirror the Claude Code hook (`vscode-extension/hook/lumi-hook.sh`): robust CLI
resolution (`LUMI_BIN` → global `lumi` → monorepo `core/dist/cli-bin.js`) and `|| true` safety so a
failure never breaks your session.

> **Prereq for all:** the `lumi` CLI must be reachable — either `npm install -g` it (global `lumi`)
> or `export LUMI_BIN=/path/to/core/dist/cli-bin.js` in the shell that launches the tool.

## Codex
Codex calls its `notify` program with the event JSON as **argv[1]** (`agent-turn-complete` carries
`last-assistant-message`). Add to **`~/.codex/config.toml`** (user-level, before any `[table]`):
```toml
notify = ["bash", "/ABS/PATH/adapters/codex/lumi-codex-hook.sh"]
```

## Gemini CLI
Gemini's `AfterAgent` hook passes JSON on **stdin** with `prompt_response`; its stdout must be a
single JSON object (the adapter prints `{"decision":"allow"}`). Register via `adapters/gemini/hooks.json`
shipped in the extension, or merge it into **`~/.gemini/settings.json`** under `hooks.AfterAgent`.
*(Known Gemini bug #15712 may skip text-only turns; for one-shot use,
`gemini -p "..." | lumi feed --source gemini` always works.)*

## Cursor
Cursor 1.7+'s `stop` hook passes JSON on **stdin** with a `transcript_path` (`.jsonl`); the adapter
reads it and feeds it. Add `adapters/cursor/hooks.json` to **`~/.cursor/hooks.json`** (or project
`.cursor/hooks.json`), pointing `command` at the script's absolute path.

## Copilot CLI
Copilot's `agentStop` hook passes JSON on **stdin** with a `transcript_path`; the adapter reads it
and feeds it. Write `adapters/copilot/hooks.json` to **`~/.copilot/hooks/lumi.json`** (point the
script path at where you copied the adapter).

## OpenCode
OpenCode plugins subscribe to its event bus. Lumi's plugin
(`claude-plugin/.opencode/plugins/lumi.js`) already injects the teaching skill; it now also listens
for the **`message.updated`** event and, once an assistant message **completes** (`info.time.completed`
set), fetches the message text via `client.session.message(...)` and pipes it to
`lumi feed --source opencode`. The feed-write is automatic once the plugin loads — no extra config.
*(Install unchanged: drop/symlink `claude-plugin/.opencode/` into OpenCode's plugin location. Set
`LUMI_BIN` to point at a specific CLI build, else it falls back to the bundled `core/dist/cli-bin.js`,
then a global `lumi`.)*

## Generic (any tool)
```sh
your-ai-tool ... | ./adapters/generic/lumi-pipe.sh <toolname>
```

## Honesty
The adapters are **verified offline** (each correctly extracts the tool's output and writes lesson
events to the feed — smoke-tested via the mock generator). What needs each tool installed on a real
machine: confirming the tool actually *fires* the hook during a live session and the feed fills as
you work. The exact hook payload shapes are sourced from each tool's docs (see the script headers).
