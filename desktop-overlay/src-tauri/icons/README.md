# Icons

## What's here now (placeholders)

- `icon.png` — a copy of the Lumi brand icon (`vscode-extension/media/icon.png`,
  512×512). Used as the window/app icon and enough for `tauri dev`.
- `icon.ico` — the same PNG wrapped into a single-entry Windows `.ico`
  (PNG-compressed, 256-class). Generated by `scripts/png-to-ico.mjs` so a
  Windows `tauri build` has an `.ico` to embed without the full icon toolchain.

These placeholders are **not** the canonical multi-resolution set. They are good
enough to dev and to attempt a build; for crisp icons at every size, regenerate.

## Regenerate the canonical set

From the `desktop-overlay/` folder, after `npm install`:

```bash
# macOS / Linux
./scripts/gen-icons.sh
# Windows
.\scripts\gen-icons.ps1
```

These run `npx @tauri-apps/cli icon <source.png>`, which produces `32x32.png`,
`128x128.png`, `128x128@2x.png`, `icon.icns`, `icon.ico`, and the Windows Store
logos here. `@tauri-apps/cli` is a prebuilt binary, so **Rust is not required**
just to generate icons — only `npm install` is.

After regenerating, you may widen the `bundle.icon` list in `tauri.conf.json` to
include the extra sizes if you want them in installers.
